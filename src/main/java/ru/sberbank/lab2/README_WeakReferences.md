Результаты:
1) Малый размер кучи, быстрый переход объектов в старшее поколение. Weak-ссылки очищаются немедленно при Minor GC, 
если объект недоступен по сильным ссылкам. Это приводит к частым Minor GC, но Full GC может быть сравнительно редким, 
так как объекты не накапливаются в старшем поколении через weak-ссылки - они быстро освобождаются
2) Включение weak-ссылок для всех объектов означает, что практически все объекты не удерживаются, 
и они будут собраны при ближайшем GC, если больше не нужны. 
Частота GC может оставаться высокой из-за постоянных аллокаций, но Full GC по-прежнему нечасты, 
так как слабые ссылки не задерживают объекты в памяти
3) Большая куча даёт больше «прокачки» через молодое поколение. Weak-ссылки очищаются практически мгновенно, 
так что нет существенного накопления объектов. Это снижает необходимость в Full GC, 
так как слабые ссылки не удерживают объект и при любом Minor GC они исчезают. 
В результате Full GC может происходить крайне редко, если вообще будет нужен...
